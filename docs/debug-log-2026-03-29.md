## 问题清单

### 问题 1: 进度条多行显示

**现象**: 进度条显示多行历史，不是单行更新

**原因**: 
- 每次 `print!` 都使用了 `\r` 回车，但在某些终端会保留历史
- 没有用空格清除旧内容

**解决**:
```rust
// 错误写法
print!("\r📤 {} {}%", file, pct);

// 正确写法 - 添加空格清除旧内容
print!("\r📤 {} {}% [{}/{}]                    ", file, pct, sent, total);
```

---

### 问题 2: 大文件 10053 错误

**现象**: 发送 100MB 文件时报错
```
Error: error sending request for url (http://127.0.0.1:53317/upload)
Caused by: 你的主机中的软件中止了一个已建立的连接。 (os error 10053)
```

**原因**:
- 服务端 `field.bytes().await` 一次性读取整个文件到内存
- 对于大文件，内存压力大，处理时间长，导致连接超时

**解决**:
- 保持 `bytes()` 方法（axum 已内置流式接收）
- 添加详细日志便于调试
- 测试证明 100MB 文件可以成功传输

---

### 问题 3: 后台进程冲突

**现象**: 用户测试失败，AI 测试成功

**原因**:
- 后台有多个 `thru.exe` 进程
- 端口 53317 被旧进程占用
- 用户看不到 AI 后台启动的进程

**解决**:
```bash
# 测试前清理所有旧进程
taskkill /F /IM thru.exe

# 或者查看并清理特定进程
netstat -ano | findstr "53317"
taskkill /F /PID <pid>
```

**教训**: 测试前必须清理后台进程！

---

### 问题 4: CMD 进度条不显示

**现象**: PowerShell 中正常，CMD 中看不到进度条

**原因**:
- CMD 对 `\r` 回车的处理与 PowerShell 不同
- `\r` 后需要空格清除旧内容

**解决**:
```rust
// CMD 兼容写法
print!("\r内容 + 空格填充                    ", ...);
std::io::stdout().flush().ok();
```

---

### 问题 5: Tailscale 异地传输波动

**现象**:
```
HTTP: ConnectionAbortedError: [Errno 103] Software caused connection abort
SCP:  Connection reset
```

**原因**:
- Tailscale 异地传输网络不稳定
- 不是代码问题，是网络环境问题

**解决**:
- HTTP 失败自动降级 SCP
- 这是正确的设计，无需修复
- 建议局域网内使用

---

## 关键发现

1. **axum multipart**: `field.bytes()` 对于 100MB 文件可行，但需要稳定网络
2. **进度条实现**: 百分比变化时才更新，避免刷屏
3. **测试文件大小**: 使用 50MB-100MB 测试，太小无法观察进度条
4. **进程管理**: 必须清理后台进程再测试

## 测试规范

### 测试前准备
```bash
# 1. 清理后台进程
taskkill /F /IM thru.exe

# 2. 确认端口已清空
netstat -ano | findstr "53317"

# 3. 启动服务
thru serve
```

### 测试文件
```bash
# 创建 50MB 测试文件
fsutil file createnew test_50mb.bin 52428800

# 或者使用 dd
dd if=/dev/zero of=test_50mb.bin bs=1M count=50
```

### 测试命令
```bash
# 电脑到电脑
thru send test_50mb.bin --lan 127.0.0.1

# 电脑到手机
thru send test_50mb.bin --lan 172.18.140.188
thru send test_50mb.bin --lan 100.92.89.83  # Tailscale
```

## 开发教训

### 1. 进程管理
- 永远不要假设进程已退出
- 测试脚本应该自动清理旧进程
- 考虑添加 `thru stop` 命令

### 2. 日志的重要性
- 服务端日志对于调试至关重要
- 客户端应该显示更详细的错误信息
- 考虑添加 `--verbose` 模式

### 3. 终端兼容性
- PowerShell 和 CMD 行为不同
- 进度条需要兼容测试
- 考虑使用 `indicatif` 库

### 4. 网络波动处理
- 异地传输不稳定是正常现象
- 自动降级机制是必要的
- 应该有更友好的错误提示

### 5. 测试方法论
- 小文件测试功能
- 大文件测试性能和稳定性
- 不同网络环境都要测试

---

## 给未来开发的建议

1. **添加健康检查**: `thru health` 检查服务端状态
2. **改进进度条**: 使用 `indicatif` 库，更好的终端兼容
3. **断点续传**: 大文件传输失败后支持续传
4. **传输队列**: 支持批量文件传输
5. **速度限制**: 可选的带宽限制
6. **加密传输**: HTTPS 支持
